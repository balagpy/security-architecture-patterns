# Contributing

Contributions are welcome when they meet this repository's research quality bar.

## Contribution Types

- New architecture case studies.
- Expanded mitigation analysis with explicit tradeoffs.
- Improved diagrams and threat models.
- Reproducible, defensive-oriented demos.

## Required Submission Standard

Every case-study folder should include:

- `README.md`
- `architecture.svg`
- `attack-flow.svg`
- `sequence.svg`
- `mitigations.md`
- `references.md`
- `diagrams/architecture.mmd`
- `diagrams/attack-flow.mmd`
- `diagrams/sequence.mmd`

## Content Quality Rules

- No filler content.
- References are required for architectural claims.
- Tradeoff analysis is mandatory for mitigation sections.
- At least one trust-boundary or sequence visual is mandatory.
- Keep tone engineering-focused and defensive.

## Before Opening a PR

- Align with [METHODOLOGY.md](./METHODOLOGY.md).
- Validate markdown readability in raw view.
- Ensure diagrams render correctly in SVG.
